# 变更提案: 计算机学习博客站点（MkDocs Material）

## 需求背景
当前仓库已包含 MkDocs Material 的基础配置，但内容目录与导航仍处于空白/未对齐状态；同时希望参考优秀博客站点的“清晰导航 + 视觉友好首页 + 强搜索 + 标签体系 + 暗亮配色”体验，在不引入额外授权/复杂依赖的前提下，落地一个可持续迭代的学习型站点框架。

## 产品分析

### 目标用户与场景
- **用户群体:** 站点作者本人（主）、偶尔浏览的同学/朋友（次）
- **使用场景:** 学习过程中随手记录、回顾与复习；对外分享课程/专题笔记；通过搜索快速定位知识点
- **核心痛点:** 内容逐渐变多后难以找回；导航不清晰导致结构混乱；缺少统一的“标签/分类”入口

### 价值主张与成功指标
- **价值主张:** 以最小依赖在 MkDocs Material 上搭建“结构化 + 可搜索 + 可扩展”的学习笔记站点
- **成功指标:** 顶部栏目可扩展；首页提供明确入口；支持暗/亮主题切换；站内全文搜索可用；标签页可作为“标签/分类”入口

### 人文关怀
避免收集用户隐私数据；站点为纯静态内容，默认不引入评论系统与追踪脚本，后续如需统计应选择透明、低侵入方案。

## 变更内容
1. 重构 `mkdocs.yml`：对齐导航结构、开启主题能力与必要 Markdown 扩展
2. 补齐 `docs/` 内容骨架：栏目首页、标签页、示例内容页
3. 首页视觉升级：欢迎区 + 快捷入口卡片 + 最近更新入口（先手动维护）

## 影响范围
- **模块:** mkdocs-config / content / deploy
- **文件:** `mkdocs.yml`、`docs/index.md`、`docs/**`、`docs/assets/**`
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 结构化导航与栏目框架
**模块:** mkdocs-config
将顶部导航固定为「主页 / 公开课 / 工具 / 语言 / 杂项 / 其他」，并允许后续新增栏目。

#### 场景: 新增内容后仍可快速定位
在内容增多的情况下，通过导航与索引页快速定位目标页面。
- 预期结果: 导航指向的文件均存在且层级清晰

### 需求: 首页清晰入口与最近更新入口
**模块:** content
首页展示站点定位、常用入口与“最近更新”区域入口。

#### 场景: 首次访问能知道从哪里开始
用户打开首页即可看到学习路线/栏目入口与更新入口。
- 预期结果: 首页包含欢迎区、精选入口卡片、最近更新入口

### 需求: 全文搜索 + 标签/分类入口
**模块:** mkdocs-config
启用搜索与 tags，使内容可检索，并以“标签前缀约定”实现分类式浏览。

#### 场景: 通过关键词找到知识点
用户输入关键词可定位到相关页面，并可通过标签页继续浏览同类内容。
- 预期结果: 搜索可用；存在 tags 索引页；页面可声明 tags 元信息

## 风险评估
- **风险:** 过度自定义导致主题升级困难
- **缓解:** 以配置与轻量 CSS 为主，避免大量模板覆盖（overrides）
