# 技术设计: 计算机学习博客站点（MkDocs Material）

## 技术方案

### 核心技术
- MkDocs
- Material for MkDocs（主题）
- 内置插件：`search`、`tags`（在“以当前配置为准”的前提下优先使用）

### 实现要点
- 调整 `mkdocs.yml`：
  - 启用顶部导航 Tabs、回到顶部、代码复制等体验性功能
  - 配置 palette 实现暗/亮主题切换
  - 配置 `search` 以支持中英文检索（以环境可用能力为准）
  - 启用 `tags` 并提供 tags 索引页
  - 增补 Markdown 扩展以支持卡片/布局（`md_in_html`、`attr_list`、`pymdownx.*`）
- 内容骨架：
  - 为每个顶级栏目提供 index 页，作为“目录入口/路线图”
  - 首页使用 Markdown + 轻量 CSS 实现欢迎区与卡片入口（不依赖 Insiders 模板）

## 架构决策 ADR

### ADR-001: 以配置优先实现博客站点框架
**上下文:** 目标是“好看又清楚”的站点体验，但约束为“以当前配置可以实现的为准”，避免引入 Insiders/复杂插件。
**决策:** 采用 Material 官方能力（导航 Tabs、palette、search、tags）+ 少量 CSS 达成首页与导航视觉。
**理由:** 依赖更少、维护成本更低、与当前仓库兼容。
**替代方案:** 引入博客增强插件 → 拒绝原因: 依赖与部署链路不确定、超出“当前配置为准”边界。
**影响:** “最近更新”模块初期可能需要手动维护，后续可再迭代为自动化。

## 安全与性能
- **安全:** 不引入服务端与敏感数据；避免在仓库提交任何 Token/密钥。
- **性能:** 站点为静态资源，重点关注构建速度与页面体积；CSS/图片保持轻量。

## 测试与部署
- **测试:** 本地运行 `mkdocs build --strict`；检查 nav 指向文件存在；抽查标签页与搜索是否工作。
- **部署:** GitHub 推送触发 Vercel 构建；确保 Vercel 构建命令与输出目录与 MkDocs 对齐。
