# deploy

## 目的
定义从本地写作到线上发布的流程与约束，确保站点可稳定构建与回滚。

## 模块概述
- **职责:** GitHub 仓库结构约定、Vercel 构建与输出约定
- **状态:** 🚧开发中
- **最后更新:** 2026-01-28

## 规范

### 部署流程
- 作者在本地编辑 `docs/` 与 `mkdocs.yml`
- 推送到 GitHub 后由 Vercel 自动构建部署

### 构建要求
- `mkdocs.yml` 中的 nav 不应指向不存在文件
- 建议开启严格构建（`mkdocs build --strict`）以尽早发现链接与配置问题
  - MkDocs 默认输出目录为 `site/`（可用于 Vercel 的输出目录设置）
