# mkdocs-config

## 目的
统一管理站点外观、导航结构、插件能力（搜索/标签/暗亮配色）与构建策略。

## 模块概述
- **职责:** 维护 `mkdocs.yml`、主题配置、Markdown 扩展与静态资源引用
- **状态:** ✅稳定
- **最后更新:** 2026-01-28

## 规范

### 主题与导航
- 顶部导航（Tabs）优先呈现核心栏目：主页 / 公开课 / 工具 / 语言 / 杂项 / 其他
- 允许后续扩展栏目，不破坏既有链接结构
  - 当前实现: `mkdocs.yml` 中以顶层 section 作为 Tabs，主页下包含「首页/最近更新/标签」

### 插件能力
- 启用全文搜索（含中文分词能力以 Material 的 search 配置为准）
- 启用 tags 作为“标签/分类”的统一入口（分类通过约定标签前缀实现）
  - 当前实现: `plugins: [search, tags]`，并指定 `tags_file: tags.md`

### 暗/亮主题
- 使用 palette 提供暗色/亮色两套配色，并提供切换按钮

### 静态资源
- 统一放置在 `docs/assets/` 下
- CSS 仅用于布局与配色增强，避免过度定制导致升级困难
  - 当前实现: `docs/assets/stylesheets/extra.css`（首页 hero 与卡片布局）
